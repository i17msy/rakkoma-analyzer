# ラッコM&A スクレイピング & フィルタリングツール

## プロジェクト概要

ラッコM&A（rakkoma.com）はオンライン事業の売買プラットフォーム。
YouTubeチャンネルも売買対象として掲載されており、匿名の形でGoogle Adsense月次収益・チャンネル登録者数・月間再生数などが確認できる。

**目的**: 新規掲載案件を自動監視し、事前定義した基準を満たすYouTubeチャンネルのみSlack通知する。

### なぜこのツールが必要か

1. MLBパイプライン構築の前段として、ラッコM&Aで「継続収益があり再現性のありそうなチャンネル」を発見した
2. この「発見 → 分析 → 判断」のプロセスを自動化・アルゴリズム化したい
3. 現状のメール通知はノイズが多く、条件に合う案件だけSlackで受け取れると理想

## サイト仕様（要調査・仮説）

### 案件ページで確認できる情報（推定）
- 事業種別（YouTube / ブログ / EC 等）

- 売却希望価格 / 年収倍率

## 設計方針

### フェーズ1: 新着監視 + Slack通知

```
定期実行（cron / cloud routine）
  └─ 一覧ページをスクレイピング
       └─ 既存IDと差分 → 新着案件を特定
            └─ フィルタリング（下記条件）
                 └─ 条件合致 → Slack通知
```

**フィルタリング条件（叩き台）**
| 条件 | 値 | 理由 |
|---|---|---|
| 事業種別 | YouTube チャンネル | 対象を絞る |
| 月次収益 | ≥ N万円（要設定） | 収益性の閾値 |
| 収益源 | Google Adsense | 再現性の指標 |
| 収益倍率（価格/年収） | ≤ X倍（要設定） | 割安感の指標 |
| 月間再生数 | ≥ M万回（要設定） | 規模感 |

### フェーズ2: 案件詳細の取得 + 構造化

- 詳細ページをスクレイピングして案件情報を構造化
- Slack通知に要約（月収・価格・倍率・チャンネル概要）を含める
- 案件DBをローカルJSONで管理（既読・スキップ・ウォッチリスト）

### フェーズ3: 自動スコアリング（将来）

- LLMで案件説明を解析し「再現性スコア」を算出
- 「解説動画系」「翻訳系」「スポーツ系」などジャンル分類
- 過去に収益が出ているチャンネルのパターンと照合

1. **ログインなしで一覧ページにアクセスできるか？**
